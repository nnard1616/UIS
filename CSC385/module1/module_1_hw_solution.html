1. (2 points) State the order of magnitude for each of the following mathematical functions.

  1. 3 + 4logN     O(logN)
  2. 5n3 - 7n + 30     O(N3)
  3. 70000000n2 + 40000n + 108     O(N2)
  4. (n3 + 3n2 + 2n) / 6     O(N3)


The goal here was to find the order of magnitude growth rate function from Table 1.1 in the lecture that corresponds to each mathematical function. The order of magnitude is determined by finding the dominant term, which is the term that contributes the most to the function's value when N is arbitrarily large, and disregarding any coefficient that term may have. The other terms of the mathematical function are ignored. For example, for the mathematical function 45n3 + 14n2 +3n + 6, the dominant term is the cubic term, 45n3, since that term will contribute most significantly to the overall value of the function when N is arbitrarily large. The quadratic, linear, and constant terms are ignored, and the coefficient of the cubic term, 45, is disregarded. This leaves n3, which corresponds to O(N3).

2. (2 points) Assume you have two algorithms, A and B, both of which perform the same function, although their implementations differ. Assume that algorithm A has a running time of O(N3 + 2N2 + 6N + 3) and algorithm B has a running time of O(6N2 + 4N + 10). Also assume that the value of N is restricted to the set of Natural numbers, including 0; i.e., {0, 1, 2, 3, ...}. If the performances of the two algorithms are compared, assuming identical test conditions for both algorithms, match each of the following questions on the left to the correct answer on the right. For this question you must look deeper than the order of magnitude values for Big-Oh.

NOTE: When doing an analysis such as this, only non-negative, integral values of N are of importance, since the notion of a fractional problem size is undefined, as is the notion of a negative problem size. A problem size of 0 represents the empty set problem, which is significant to an analysis of the efficiencies of two algorithms.

  1. Which algorithm would be expected to be most efficient when N is arbitrarily large; i.e., as N approaches infinity?

    Algorithm B. To answer this question, you only need to consider the order of magnitude of the two algorithms, since for a large enough value of N, the highest order term will be dominant. Algorithm B has an order of magnitude of O(N2), whereas the order of magnitude for algorithm A is O(N3). When N reaches a sufficiently high value (N = 4 in this case), the cubic term of algorithm A will ensure that for all values greater than or equal N, the value of algorithm A will always be greater than the value of algorithm B. Since lower values correspond to higher efficiency, Algorithm B will be more efficient when N is large.

    To find the precise value that tips the balance towards one algorithm or the other, you can set up an inequality between the two polynomials and solve for N, or you can create a table where each polynomial is evaluated for a series of Natural numbers, and look for the value at which the growth of one polynomial clearly will always be greater than the growth of the other polynomial    .

  2. Which algorithm would be expected to be most efficient when N is arbitrarily small; i.e., as N approaches 0?

    Algorithm A. For this question, you already know that algorithm A is more efficient when N is arbitrarily large, and it may be more efficient for all values of N. To test the hypothesis that algorithm A is more efficient for all values of N, you need to find out whether there is some N > 0 such that N3 + 2N2 + 6N + 3 < 6N2 + 4N + 10. If so, you can disprove the hypothesis by contradiction; if not, the hypothesis holds, and algorithm A will be more efficient even when N is arbitrarily small. It turns out in this case that algorithm A is more efficient than algorithm B for N < 3, thus disproving the hypothesis by contradiction.
