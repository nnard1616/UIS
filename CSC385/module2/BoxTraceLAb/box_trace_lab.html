Module 2: Box Trace Lab

This lab will help you become familiar with using the box trace method to trace a recursive algorithm.
 
Introduction

Tracing the execution of a recursive algorithm can be tricky.  The difficulty lies in the fact that you need to keep track of multiple instances of the recursive method.  One thing that is important to remember is that when a recursive call is made, execution of the current instance of the method is suspended and a new instance of the recursive method is created.  New method instances are created until the base case is reached.  Once the base case has been reached, and a concrete value can be returned, execution traces back through the method instances in reverse order until the original recursive call can return a value.

Commonly a stack is used to illustrate this process.  An alternative method of tracing a recursive algorithm is known as the Box Trace method.  This method is sort of like a stack placed on its side, but it uses arrows to indicate the flow of execution.
